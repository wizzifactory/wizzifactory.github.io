Code

In this section you can browse the ittf source code of kernel modules and default plugins.

Since version 0.3 the wizzi factory generates itself, so browsing the source can help the understanding of the documentation.

Ittf notation legend

  1. nodename nodevalue (rest of line)
  2.   $nodecommand parameters
  3.   mixinCall( param-1, param-2, ...
  • 'nodename' in green is a semantic node name. The meaning of nodename is given by the wizzi schema, if the ittf document is schemed, otherwise it is a free meaning, given by the document author.
  • The rest of the line in white is the value of 'nodename'.
  • '$nodecommand' is a template command. Node commands are documented here
  • 'mixinCall' in red is a fragment inclusion command. In the code browser it is clickable and a modal dialog with the content of the mixin is shown. The closing parenthesis is optional.

Kernel packages

wizzi The core package of the wizzi factory.
wizzi-mtree Loader for wizzi indented text tree format (ittf) documents.
wizzi-repo Repository manager for the wizzi factory.
wizzi-utils Utility library for the wizzi factory.

Default plugin packages

wizzi-core A Wizzi Factory plugin with core schemas and artifact generators.
wizzi-js A Wizzi Factory plugin targeting the javascript language.
wizzi-web A Wizzi Factory plugin targeting common web technologies (html, css, scss, markdown, graphql).

The demo package

wizzi-demo The most complete set of examples of the wizzi factory.
