The code section allow the browsing of the kernel modules of the wizzi factory and two core plugins.
Since version v3 (0.3.x pre-release) the source of the wizzi factory code is made of wizzi models, so browsing the source helps to better understand thing explained by the documentation.
Ittf notation legend
  1. nodename nodevalue
  2.   $nodecommand parameters
  3.   mixinCall( param-1, param-2, ... )
'nodename' in green is a semantic node name. The meaning of nodename is given by the wizzi schema, if the ittf document is schemed, or is a free meaning, given by the document author.
