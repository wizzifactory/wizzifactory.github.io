Main Wizzi Factory Roadmap

Todos

Implement a line continuation character

Long node values looks better when in a new line. the blackslash at the beginning of the line should be fine. At the beginning of the continuation line, not at the end of the continued line, is more Wizzi style.

Implement multi node mixin call Ittf statement in the Ittf prettifier
Manage the close paren of mixin call Ittf statement in the Ittf prettifier
Correctly implement comments in the Ittf prettifier
Add node documentation support to the Ittf prettifier

The Ittf prettifier could be implementaed also as a Wizzi model loader feature.

Ittf nodes could be beautified and have a popup linked to the Wizzi schema documentation of the node semantics.

Have a look at http://redcloth.org/textile for ideas evolving the @docs@ Wizzi schema.
Implement an is-root attribute on a the WizziMetaSchema Element element

Allow the first element on the schema not be the root element

Implement the xml-tag attribute on the WizziMetaSchema Attribute element
In js-wizzi-schema the throw node should accept child arguments

Issues

IttfInclude. $param root node in included document

If the root node of the included document is a $params node the loading must fail and the error must be logged

Error messages sometimes are obscure or incorrect

We need a better strategy to print the exact file, and line of code where an error occurred

IttfLoader - Mixins. Error loop when passing object arguments to a mixin.

When a mixin has an object param and calls another mixin passing an object argument the loading loops

The role of the require element of the wizzischema is unclear

Should be node-js require only or may be used to require features.
