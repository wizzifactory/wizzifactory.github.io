Getting started

You have to be really motivated, beyond what I can imagine, for wanting to start using the wizzi factory.

I don't want to incourage you making you believe that you can quickly get started and run. Accordingly this getting started section will be quite scarce.

At the current time the main problem with the wizzi factory is error detection. You can get an error that does not clearly show where it has occurred.

So the only way to get started is dive into code and help me to implement a better error tracking.

But i don't want to let the brave and the reckless without an option on where to start.

The package wizzi-demo, that can be installed from npm, is the playground where we are concentrating our efforts to make the wizzi factory usable beyond its main purpose, that, you should remember, is the generation of its next improved version.

If you dare to try playing with the wizzi-demo we promise to do our best to give help.
