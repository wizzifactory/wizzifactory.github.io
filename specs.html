The WizziModel and WizziSchema notation

This IttfDocument ...

  1. rdbms School
  2.     table Student
  3.         column Id
  4.             type integer
  5.         column Name
  6.             type string
  7.     table Teacher
  8.         column Id
  9.             type integer
  10.         column Name
  11.             type string

... could be validated against the following WizziSchema:

  1. wizzischema rdbms
  2.     e rdbms
  3.         r table/s
  4.     e table/s
  5.         r column/s
  6.     e column/s
  7.         a type
  8.             restrict 
  9.                 enum string
  10.                 enum boolean
  11.                 enum integer
  12.                 enum float
  13.                 enum date
  1. wizzischema schema-name
  2.      
  3.     [0..n]require module-name
  4.         declare-var var-name
  5.      
  6.     [0..3]export-to json|xml|prettify
  7.      
  8.     [0..n]element element-name [: super-element-name]
  9.         super super-element-name
  10.         tag tag-string
  11.         xml-tag xml-tag-string
  12.         is-abstract 
  13.         is-root 
  14.         name-is-required 
  15.         add-to-children 
  16.         [0..n]attributes 
  17.         [0..n]relations 
  18.         [0..n]methods

Relation element

Format:

  1. ... 
  2.     r related-element-name[ [plural-suffix-a] / plural-suffix-b ]
  3.         one-to-one 
  4.         no-get

Example:

  1. ... 
  2.     e table : dbitem
  3.         r column/s
  4.         r propert-y/ies
  5.         r displayLayout
  6.             one-to-one

related-element-name plus the eventual plural-suffix-a must give the name of an Element element defined in the schema (that is the related Element); in the example: column, property, displayLayout.

Attribute element

Describes an attribute of an Element element.

Format:

  1. ... 
  2.     a attribute-name
  3.         tag tag-string
  4.         xml-tag xml-tag-string
  5.         type type-name
  6.         default default-value
  7.         default-when-declared default-value-when-declared
  8.         is-datatype 
  9.         is-required 
  10.         [0..n]restricts

Restrict element

Format:

  1. ... 
  2.     restrict 
  3.         [0..n]enum enum-value
  4.         max-length value
  5.         min-length value
  6.         max-value value
  7.         min-value value
  8.         regexp value

Model type code elements

The following elements of the WizziMetaSchema consent the insertion of javascript code in the WizziModelType that will be generated by the WizziModelProduction

ExportTo element

Format:

  1. wizzischema schema-name
  2.     export-to format-name

Corrently available format is: 'json'.

En exportTo element causes the insertion in the WizziModelType of the WizziSchema of the prototype functions for exporting the tree data structure of the model in the requested format.

Require element

Format:

  1. wizzischema schema-name
  2.     require module-name
  3.         declare-var var-name
  4.         invoke invoke-string
  5.         kind kind-name

Method element

Declares a method that will be added in the WizziModelType to the pseudo-class of the containing Element. The method can access and manipulate the entire model tree structure.

Format:

  1. ... 
  2.     m method-name
  3.         [0..n]params 
  4.         [0..n]statements

Parameter element

Format:

  1. ... 
  2.     param param-name

Statement element

The statement element uses a subset of the CodeWizziSchema 'js'

IttfDocuments search path

When IttfDocuments are included or mixed-in, given {path current-dir{path} as the path of the including or mixing IttfDocument, they are searched (in order):

  • in the {path current-dir{path} folder;
  • in the {path current-dir{path}/t folder;
  • in the folder named {path t{path} of the parent folder of the {path current-dir{path};
  • in the folder named {path t{path} of the ancestor folders of the {path current-dir{path}.

Given this folder structure

  1. folder-a 
  2.     t 
  3.         layout.html.ittf 
  4.     folder-b 
  5.         t 
  6.             dialog.html.ittf 
  7.         my-home-page.html.ittf

The IttfDocument my-home-page.html.ittf could be:

  1. layout( home )
  2.     $include dialog( )

Js-Wizzi, the template engine language

The WizziModelType structure

To use WizziModelInstances as object contexts in template transformations one must get familiar with the structure of a WizziModelType.

Each schema element is mapped to a javascript pseudo class that inherits from a super element or an auto-generated base element.

Example WizziSchema

  1. wizzischema rdbms
  2.     e dataitem
  3.         a sqlname
  4.         e comment
  5.             tag #
  6.     e rdbms : dataitem
  7.         is-root 
  8.         r table/s
  9.     e table : dataitem
  10.         r column/s
  11.     e column : dataitem
  12.         a type
  13.             restrict 
  14.                 enum string
  15.                 enum boolean
  16.                 enum integer
  17.                 enum float
  18.                 enum date

Example WizziModelType

  1. /*
  2.     rdbmsBase class, standard autogenerated base class
  3. */
  4. rdbmsBase = function(name, sourceLineInfo) 
  5.     this.wzName = name || '';
  6.     this.wzParent = null;
  7.     this.wzSourceLineInfo = sourceLineInfo;
  8. {}
  9. ...
  10. /*
  11.     dataitem class
  12. */
  13. dataitem = function(name, sourceLineInfo) 
  14.     rdbmsBase.call(this, name, sourceLineInfo);
  15.     this.wzElement = "dataitem";
  16.     this.sqlname = null;
  17.     // relation comment
  18.     this.comments = [];
  19. {}
  20. util.inherits(dataitem, rdbmsBase);
  21. ...
  22. /*
  23.     table class
  24. */
  25. table = function(name, sourceLineInfo) 
  26.     dataitem.call(this, name, sourceLineInfo);
  27.     this.wzElement = "table";
  28.     // relation column
  29.     this.columns = [];
  30. {}
  31. util.inherits(table, dataitem);
  32. ...

Model objects properties that a template developer can use:

wzElement Is the name of the element corresponding to the IttfNodeName of the source IttfNode
wzName Is the IttfNodeValue of the source IttfNode
wzParent Is the reference to the parent of the source IttfNode
wzSourceLineInfo Data info of the source IttfNode: row, col, sourceKey
attributes Each attribute of the schema element becomes a simple type property. In the example: sqlname, type, ...
collections Each related one-to-many element of the schema element becomes an array property. In the example: comments, tables, columns
objects Each related one-to-one element of the schema element becomes an object property.

Example template

  1. html 
  2.     body 
  3.         ul Schema tables
  4.             $foreach table in rdbms.tables
  5.                 li ${table.wzName}
