Getting started

                        

                            
You have to be really motivated, for wanting to start using the wizzi factory.
I don't want to incourage you saying that one can quickly get started and run. Accordingly this getting started section will be quite scarce.
At the current time the main problem with the wizzi factory is error detection. You can get an error that does not clearly show where it has occurred.
So the only way to get started is dive into code and help us to implement a better error tracking.
But i don't want to let the brave without an option on where to start.
The package wizzi-demo, that can be installed from npm, is the playground where we are concentrating our efforts to make the wizzi factory usable beyond its declared purpose, that is the generation of its next improved version.
If you dare to play with the wizzi-demo we promise to do our best to give help.

                        
Wizzi-demo starter package

                        

                            
Overview
The wizzi demo package is made of samples and links to the project documentation.
Have a look at the    github project    .    
The readme file in each folder give you the links to concept, documentation, api and source code, relative to the samples in that folder.
Sections
Sample folders cover these areas:
        
  • mtree: ittf documents and mTrees    
  • jswizzi    
  • wizzi models    
  • wizzischema: wizzi schemas and wizzi model types    
  • schema languages    
  • wizzi jobs    
  • wizzi plugins: creating and using a local sample plugin    
The wizzi plugins section contains samples of model transformers and artifact generators.
Installation
    
        # Clone this repository        
            $ git clone https://github.com/wizzifactory/wizzi-demo            
                
                    # Go into the repository                    
                        $ cd wizzi-demo                        
                            
                                # Install dependencies                                
                                    $ npm install                                    
                                    
Running samples                                    
                                        
                                            # Run samples in folders:                                            
                                                $ node mtree                                                
                                                    $ node wizzimodel                                                    
                                                        $ ...                                                                                                            
Build your samples                                                    
This is left to you.
